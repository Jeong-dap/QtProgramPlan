## 1. 프로젝트 개요

### 1.1 배경 및 목적

Qt C++ 프레임워크를 활용하여 데스크탑 환경에서 동작하는 고객관리 프로그램을 개발한다.  
사용자가 고객 정보를 직관적으로 추가·조회·수정·삭제할 수 있는 GUI 기반 애플리케이션을 목표로 한다.

### 1.2 핵심 목표

- Qt Widgets 기반의 직관적인 GUI 제공 (GUI 완성도 중심 평가)
- 고객 데이터의 CRUD(Create, Read, Update, Delete) 기능 완전 구현
- JSON 파일 기반 데이터 영속성 보장 (외부 DB 미사용)
- Windows 10+ 환경에서 안정적으로 동작하는 애플리케이션 완성

### 1.3 평가 기준

본 프로젝트는 **GUI 이해도 및 완성도** 중심으로 평가된다.

- Qt 위젯 레이아웃의 적절한 구성 및 사용성
- CRUD 기능의 UI 흐름이 직관적이고 자연스러운가
- 입력 유효성 처리 및 사용자 피드백 (오류 메시지 등)
- 전체적인 화면 완성도 (깨지는 레이아웃 없이 깔끔하게 동작)

> ⚠️ 알고리즘 최적화나 고급 설계 패턴보다 **동작하는 GUI**를 우선한다.

## 5. 데이터 구조 설계

### 5.1 Customer 클래스 (C++)

```cpp
class Customer {
public:
    int         id;           // 고유 ID
    QString     name;         // 이름
    QString     phone;        // 연락처
    QString     email;        // 이메일
    QString     address;      // 주소
    QString     grade;        // 등급 (신규 / 일반 / VIP)
    QString     memo;         // 메모
    QDateTime   createdAt;    // 등록 시각
    QDateTime   updatedAt;    // 수정 시각
};
```

### 5.2 JSON 파일 구조

- **저장 위치:** 앱 실행 경로 (`customers.json`)
- **Qt 모듈:** `QJsonDocument`, `QJsonObject`, `QJsonArray`, `QFile`

```json
{
  "customers": [
    {
      "id": 1,
      "name": "홍길동",
      "phone": "010-1234-5678",
      "email": "hong@email.com",
      "address": "서울시 강남구 테헤란로 123",
      "grade": "VIP",
      "memo": "장기 우수 고객, 분기별 미팅 필요",
      "createdAt": "2026-01-15T09:00:00",
      "updatedAt": "2026-04-10T14:30:00"
    },
    {
      "id": 2,
      "name": "김철수",
      "phone": "010-9876-5432",
      "email": "kim@email.com",
      "address": "경기도 성남시 분당구 판교로 45",
      "grade": "일반",
      "memo": "",
      "createdAt": "2026-03-01T10:00:00",
      "updatedAt": "2026-03-01T10:00:00"
    }
  ],
  "lastId": 2
}
```

> **`lastId`** 필드: DB의 AUTO_INCREMENT를 대체하여 새 고객 등록 시 ID를 순차 부여하는 데 사용

#### JSON 파일 입출력 핵심 코드 패턴

```cpp
// 읽기
QFile file("customers.json");
file.open(QIODevice::ReadOnly);
QJsonDocument doc = QJsonDocument::fromJson(file.readAll());
QJsonArray arr = doc.object()["customers"].toArray();

// 쓰기
QJsonDocument saveDoc(rootObject);
file.open(QIODevice::WriteOnly);
file.write(saveDoc.toJson(QJsonDocument::Indented));
```

## 6. CRUD 기능 명세

### 6.1 Create (고객 등록)

- **진입점:** 메인 화면 `[+ 고객 추가]` 버튼
- **동작 흐름:**
    1. `AddCustomerDialog` 오픈
    2. 사용자가 이름, 연락처, 이메일, 주소, 등급, 메모 입력
    3. `[저장]` 클릭 → 유효성 검사
    4. `CustomerRepository::create()` 호출 → JSON 파일에 항목 추가 후 저장
    5. 목록 새로고침
- **유효성 검사 규칙:**
    - 이름: 필수, 최대 50자
    - 연락처: 필수, 숫자/하이픈 형식 (`000-0000-0000`)
    - 이메일: 선택, 입력 시 `@` 포함 형식 검사

```cpp
// 예시 시그니처
void CustomerRepository::create(const Customer& customer);
```

### 6.2 Read (고객 조회)

- **전체 조회:** 앱 시작 시 JSON 파일 로드 후 전체 목록 표시
- **키워드 검색:** 검색창 입력 시 메모리에 로드된 목록에서 이름·연락처 포함 검색
- **상세 조회:** 목록 행 클릭 시 우측 상세 패널에 정보 표시

```cpp
QList<Customer> CustomerRepository::findAll();
QList<Customer> CustomerRepository::search(const QString& keyword);
Customer        CustomerRepository::findById(int id);
```

### 6.3 Update (고객 정보 수정)

- **진입점:** 목록에서 항목 선택 후 `[✏ 수정]` 버튼 또는 더블클릭
- **동작 흐름:**
    1. 기존 데이터를 `EditCustomerDialog`에 바인딩
    2. 사용자가 항목 수정
    3. `[저장]` 클릭 → 유효성 검사
    4. `CustomerRepository::update()` 호출 → 메모리 목록 갱신 후 JSON 파일 전체 저장
    5. `updatedAt` 자동 갱신, 상세 패널 즉시 반영

```cpp
void CustomerRepository::update(const Customer& customer);
```

### 6.4 Delete (고객 삭제)

- **단건 삭제:** 항목 선택 후 `[🗑 삭제]` 버튼
- **확인 다이얼로그:** `QMessageBox::question` 으로 재확인
- **처리:**
    1. 사용자 확인
    2. `CustomerRepository::remove(int id)` 호출 → 메모리 목록에서 제거 후 JSON 파일 전체 저장
    3. 목록 새로고침, 상세 패널 초기화

```cpp
void CustomerRepository::remove(int id);
void CustomerRepository::removeSelected(const QList<int>& ids);
```

## 9. 리스크 및 고려사항

### 9.1 기술적 리스크

|리스크|대응 방안|
|---|---|
|Qt 버전 호환성|Qt 5.15 LTS 기준으로 개발 (Windows에서 안정적)|
|JSON 파일 손상|저장 전 임시 파일에 쓰고 교체하는 방식으로 원자적 저장 구현|
|대용량 데이터 성능|앱 시작 시 전체 로드 후 메모리에서 처리, 수백~수천 건 수준에서 충분|
|일정 지연|Day 2 종료 시점에 CRUD 미완성이면 권장 기능 전부 포기하고 핵심만 완성|

### 9.2 설계 원칙

- **MVC 패턴 준수:** Model(데이터/JSON), View(Qt 위젯), Controller(비즈니스 로직) 분리
- **Repository 패턴:** JSON 파일 입출력 코드를 `CustomerRepository` 클래스로 캡슐화
- **Signal & Slot:** Qt의 Signal/Slot 메커니즘을 적극 활용한 이벤트 처리

### 9.3 추후 확장 고려사항

- Windows 배포 패키지 구성 (`windeployqt` 활용)
- 거래 이력 관리 기능 (고객별 구매/상담 기록)
- 고객 데이터 엑셀(CSV) 내보내기 / 가져오기
