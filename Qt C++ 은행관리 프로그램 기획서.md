## 1. 프로젝트 개요

### 1.1 배경 및 목적

Qt C++ 프레임워크를 활용하여 데스크탑 환경에서 동작하는 은행관리 프로그램을 개발한다.  
계좌 정보 및 입출금 거래 내역을 직관적으로 등록·조회·수정·삭제할 수 있는 GUI 기반 애플리케이션을 목표로 한다.

### 1.2 핵심 목표

- Qt Widgets 기반의 직관적인 GUI 제공 (GUI 완성도 중심 평가)
- 계좌 및 거래 데이터의 CRUD(Create, Read, Update, Delete) 기능 완전 구현
- JSON 파일 기반 데이터 영속성 보장 (외부 DB 미사용)
- Windows 10+ 환경에서 안정적으로 동작하는 애플리케이션 완성

### 1.3 평가 기준

본 프로젝트는 **GUI 이해도 및 완성도** 중심으로 평가된다.

- Qt 위젯 레이아웃의 적절한 구성 및 사용성
- CRUD 기능의 UI 흐름이 직관적이고 자연스러운가
- 입력 유효성 처리 및 사용자 피드백 (오류 메시지 등)
- 전체적인 화면 완성도 (깨지는 레이아웃 없이 깔끔하게 동작)

> ⚠️ 알고리즘 최적화나 고급 설계 패턴보다 **동작하는 GUI**를 우선한다.

## 5. 데이터 구조 설계

### 5.1 Account 클래스 (C++)

```cpp
class Account {
public:
    int         id;           // 고유 ID
    QString     accountNumber;// 계좌번호
    QString     ownerName;    // 예금주
    QString     accountType;  // 계좌 유형 (입출금 / 적금 / 대출)
    long long   balance;      // 잔액 (원 단위, 정수)
    QDateTime   createdAt;    // 등록 시각
    QDateTime   updatedAt;    // 수정 시각
};
```

### 5.2 Transaction 클래스 (C++)

```cpp
class Transaction {
public:
    int         id;           // 고유 ID
    int         accountId;    // 연결된 계좌 ID
    QString     type;         // 거래 유형 (입금 / 출금)
    long long   amount;       // 거래 금액 (원 단위, 양수)
    long long   balanceAfter; // 거래 후 잔액
    QString     memo;         // 메모
    QDateTime   createdAt;    // 거래 시각
};
```

### 5.3 JSON 파일 구조

- **저장 위치:** 앱 실행 경로 (`bank.json`)
- **Qt 모듈:** `QJsonDocument`, `QJsonObject`, `QJsonArray`, `QFile`

```json
{
  "accounts": [
    {
      "id": 1,
      "accountNumber": "110-1234-5678",
      "ownerName": "홍길동",
      "accountType": "입출금",
      "balance": 1500000,
      "createdAt": "2026-01-10T09:00:00",
      "updatedAt": "2026-04-20T10:00:00"
    }
  ],
  "transactions": [
    {
      "id": 1,
      "accountId": 1,
      "type": "입금",
      "amount": 500000,
      "balanceAfter": 1500000,
      "memo": "급여 입금",
      "createdAt": "2026-04-20T10:00:00"
    },
    {
      "id": 2,
      "accountId": 1,
      "type": "출금",
      "amount": 100000,
      "balanceAfter": 1000000,
      "memo": "생활비",
      "createdAt": "2026-04-18T14:30:00"
    }
  ],
  "lastAccountId": 1,
  "lastTransactionId": 2
}
```

> **`lastAccountId` / `lastTransactionId`** 필드: DB의 AUTO_INCREMENT를 대체하여 새 항목 생성 시 ID를 순차 부여하는 데 사용

#### JSON 파일 입출력 핵심 코드 패턴

```cpp
// 읽기
QFile file("bank.json");
file.open(QIODevice::ReadOnly);
QJsonDocument doc = QJsonDocument::fromJson(file.readAll());
QJsonArray accounts = doc.object()["accounts"].toArray();
QJsonArray transactions = doc.object()["transactions"].toArray();

// 쓰기
QJsonDocument saveDoc(rootObject);
file.open(QIODevice::WriteOnly);
file.write(saveDoc.toJson(QJsonDocument::Indented));
```

## 6. CRUD 기능 명세

### 6.1 Create (계좌 등록 / 거래 등록)

#### 계좌 등록

- **진입점:** 메인 화면 `[+ 계좌 추가]` 버튼
- **동작 흐름:**
    1. `AddAccountDialog` 오픈
    2. 사용자가 계좌번호, 예금주, 유형, 초기 잔액 입력
    3. `[저장]` 클릭 → 유효성 검사
    4. `AccountRepository::create()` 호출 → JSON 파일 저장
    5. 계좌 목록 새로고침
- **유효성 검사 규칙:**
    - 계좌번호: 필수, 중복 불가, 숫자/하이픈 형식
    - 예금주: 필수, 최대 30자
    - 초기 잔액: 0 이상의 정수

#### 거래 등록 (입금 / 출금)

- **진입점:** 계좌 선택 후 `[💰 입금]` 또는 `[💸 출금]` 버튼
- **동작 흐름:**
    1. `TransactionDialog` 오픈 (선택된 계좌 정보 및 현재 잔액 표시)
    2. 거래 유형, 금액, 메모 입력
    3. `[확인]` 클릭 → 유효성 검사
    4. 잔액 업데이트 + `Transaction` 레코드 생성 → JSON 저장
    5. 계좌 목록 및 거래 내역 새로고침
- **유효성 검사 규칙:**
    - 금액: 필수, 1 이상의 정수
    - 출금 시: 잔액 >= 출금 금액 (부족 시 `QMessageBox::warning` 표시)

```cpp
void AccountRepository::create(const Account& account);
void TransactionRepository::create(const Transaction& transaction);
```

### 6.2 Read (계좌 및 거래 내역 조회)

- **전체 계좌 조회:** 앱 시작 시 JSON 파일 로드 후 전체 목록 표시
- **거래 내역 조회:** 계좌 행 클릭 시 하단 패널에 해당 계좌의 거래 내역 표시 (최신순)
- **키워드 검색:** 검색창 입력 시 계좌번호 · 예금주 포함 검색

```cpp
QList<Account>     AccountRepository::findAll();
QList<Account>     AccountRepository::search(const QString& keyword);
Account            AccountRepository::findById(int id);
QList<Transaction> TransactionRepository::findByAccountId(int accountId);
```

### 6.3 Update (계좌 정보 수정)

- **진입점:** 계좌 선택 후 `[✏ 수정]` 버튼 또는 더블클릭
- **수정 가능 항목:** 예금주, 계좌 유형 (계좌번호·잔액은 직접 수정 불가 — 거래를 통해서만 변경)
- **동작 흐름:**
    1. 기존 데이터를 `EditAccountDialog`에 바인딩
    2. 사용자가 항목 수정
    3. `[저장]` 클릭 → 유효성 검사
    4. `AccountRepository::update()` 호출 → JSON 파일 전체 저장
    5. `updatedAt` 자동 갱신

```cpp
void AccountRepository::update(const Account& account);
```

### 6.4 Delete (계좌 삭제)

- **진입점:** 계좌 선택 후 `[🗑 삭제]` 버튼
- **확인 다이얼로그:** `QMessageBox::question` 으로 재확인
- **처리:**
    1. 사용자 확인
    2. `AccountRepository::remove(int id)` 호출 → 계좌 및 연결된 거래 내역 모두 제거 → JSON 저장
    3. 계좌 목록 새로고침, 하단 거래 내역 패널 초기화

```cpp
void AccountRepository::remove(int id);         // 계좌 + 연결 거래 내역 함께 삭제
void TransactionRepository::removeByAccountId(int accountId);
```

## 9. 리스크 및 고려사항

### 9.1 기술적 리스크

|리스크|대응 방안|
|---|---|
|Qt 버전 호환성|Qt 5.15 LTS 기준으로 개발 (Windows에서 안정적)|
|JSON 파일 손상|저장 전 임시 파일에 쓰고 교체하는 방식으로 원자적 저장 구현|
|잔액 정합성 오류|입출금 시 계좌 잔액 업데이트와 거래 기록 생성을 하나의 함수 안에서 처리|
|금액 오버플로우|잔액 및 금액 필드는 `long long` 타입 사용 (최대 약 92조 원)|
|일정 지연|Day 2 종료 시점에 CRUD 미완성이면 권장 기능 전부 포기하고 핵심만 완성|

### 9.2 설계 원칙

- **MVC 패턴 준수:** Model(데이터/JSON), View(Qt 위젯), Controller(비즈니스 로직) 분리
- **Repository 패턴:** JSON 파일 입출력 코드를 `AccountRepository` / `TransactionRepository` 클래스로 캡슐화
- **Signal & Slot:** Qt의 Signal/Slot 메커니즘을 적극 활용한 이벤트 처리
- **잔액 불변 원칙:** 잔액은 직접 수정하지 않고 반드시 입금/출금 거래를 통해서만 변경

### 9.3 추후 확장 고려사항

- Windows 배포 패키지 구성 (`windeployqt` 활용)
- 계좌 간 이체 기능
- 거래 내역 CSV 내보내기
- 월별 수입/지출 통계 차트 (`QChart` 모듈 활용)
